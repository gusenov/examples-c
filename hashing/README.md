# Хэширование (hashing), преобразование ключей

## Задача

Задан набор элементов характеризующихся ключом.
Ключ определяет отношение порядка.
Как организовать данные чтобы с наименьшими усилиями извлекать элемент по ключу?

## Решение

Доступ к каждому элементу в памяти осуществляется указанием адреса.
Задача сводится к отображению ```H``` ключей (```K```) в адреса (```A```): ```H: K -> A```.

- Это отображение можно реализовать с помощью алгоритмов поиска в списках и деревьях.
- В методе *преобразования ключей* данные организуются с помощью массива. Ключи преобразуются в индексы массива.

Трудности:

Множество возможных значений ключей ГОРАЗДО больше чем множество доступных адресов в памяти (индексов массива).

Хэш-функция ```H``` отображает несколько значений аргументов в одно значение индекса.

Операция поиска:

1. вычисление индекса: ```h = H(k)```
2. проверка на *коллизию*: ```T[H(k)].key = k```
3. *разрешение коллизии*, *пробы* (probe): ```h'```, ```h''```

### Требования к хэш-функции

**Равномерное распределение ключей по всему диапазону значений индекса.**

```
r - row
c - column

[key_11]     ...    [key_1c]
...          ...    ...
[key_r1]     ...    [key_rc]

\/           \/     \/

[index_1]    ...    [index_n]
```

**Случайное распределение.**
Hashing - превращать в фарш, мешанина.

**Эффективное вычисление.**
Небольшое число основных арифметических операций.

### Выбор хэш-функции

#### ```H(k) = ORD(k) MOD N```

- ORD(k) - порядковый номер ключа k во множестве всех возможных ключей
- индекс массива i принимает значения 0 .. N-1
- N - размер массива

Плюсы:

- Равномерное распределение ключей по диапазону индексов.
- Быстрое вычисление, если N есть степень 2. Полная операция деления заменяется простым отбрасыванием двоичных цифр.

Минусы:

- Следует избегать случая N есть степень 2, если все ключи не равно вероятны, лишь немного отличающиеся ключи, будут с большей вероятностью отображаться на одно и то же значение индекса, так что получится весьма неоднородное распределение. Поэтому рекомендуется в качестве значения N выбирать простое число. Придется использовать полную операцию деления, но это не является проблемой на большинстве современных компьютеров, имеющих встроенную инструкцию деления.

#### Логические операции, исключающее "или"

Применяются к некоторым частям ключа, представленного как последовательность двоичных цифр.

Плюсы:

- На некоторых компьютерах эти операции могут выполняться быстрее, чем деление.

Минусы:

- Удивительно неоднородное распределение ключей по диапазону индексов.
